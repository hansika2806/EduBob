IBM Bob was used as the core engine of EduBob at every stage of development and as the primary AI capability within the product itself.

During development, we used Bob Plan mode to generate the full implementation plan before writing any code. We used Advanced mode to validate and refine our plan. We used Bob Code mode to implement backend and frontend phase by phase. We used Bob Ask mode to understand code structure and refine logic.

Within the EduBob product itself, Bob is used in four ways:

1. Plan mode generates assignment specifications.
2. Ask mode analyzes GitHub repositories and explains them.
3. Ask mode reviews student code against specifications and gives feedback.
4. Bob helps validate solutions using test cases.

IBM watsonx.ai is used to analyze patterns across student submissions for the dashboard.
